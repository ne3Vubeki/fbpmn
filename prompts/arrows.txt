#1
Доработка алгоритма рисования связей в тайлах между узлами.
Расположение логики расчета связей: lib\src\services\arrow_manager.dart

На данный момент следующий алгоритм рисования:
- в методе calculateConnectionPointsForSideCalculation определяются размещения узлов относительно друго друга
- в зависимости от расположения определяются точки начала и окончания связи
- далее в _createSimpleOrthogonalPath в зависимости от типа расположения начала и конца связи рисуется кривая
- существует 3 типа кривых: 1. прямая, 2. с одним изгибом, 3. с двумя изгибами

На данный момент связи могут проходить через узлы, что нарушает внешний вид и понимание общей схемы.

Тебе нужно добавить новую логику после выполнения calculateConnectionPointsForSideCalculation которая будет:
1. определять наличие на пути других узлов из списка узлов, которые будут рассчитаны
Для расчета списка узлов, которые будут проверяться на пересечение, используй расчет области размещение узла источника и цели по их крайним координатам.
Затем по этой области найди тайлы с которыми она пересекается.
Затем извлеки из выбранных тайлов из свойства nodes узлы, которые размещены в этих тайлах и сложи их в общий список.
Этот список используй для расчетов пересечений расчитываемой связи.
2. перебирать варианты начальных и конечных точек связи, чтобы связь не пересекала другой узел/узлы
3. если это не возможно, оставлять вариант из calculateConnectionPointsForSideCalculation

calculateConnectionPointsForSideCalculation и _createSimpleOrthogonalPath не исправлять.

Используй в чате только русский язык
